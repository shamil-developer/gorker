# gorker

Создать легковесный внутренний компонент (пакет worker), который заменит необходимость во внешних библиотеках типа gocron и обеспечит безопасный запуск фоновых процессов.

## Интерфейс пакета

* RunPeriodic: Запуск задачи с заданным интервалом (time.Duration) с использованием time.Ticker.
* RunContinuous: Запуск задачи, которая должна работать постоянно (например, чтение из канала или сокета), до отмены контекста.
* RunOnce: Асинхронный запуск разовой задачи.
* Wait/Shutdown: Блокирующий метод для ожидания завершения всех запущенных задач (интеграция с sync.WaitGroup).

## Дополнительный функционал

* Panic Recovery: Каждая запускаемая горутина должна быть обернута в recover(). В случае паники Runner должен залогировать Stacktrace через zap.Logger на уровне Error и корректно высвободить ресурсы.
* Управление контекстом: Механизм должен строго следовать жизненному циклу переданного context.Context. При отмене контекста все циклы/тикеры должны немедленно прекращаться.
* Логирование жизненного цикла: Запись событий "Task started", "Task finished", "Task canceled by context", "Task panicked".
* Тестирование: Компонент должен быть полностью покрыт юнит-тестами.
